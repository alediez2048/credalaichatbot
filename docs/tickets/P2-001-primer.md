# P2-001 — Document upload with file validation

**Priority:** P2
**Estimate:** 3 hours
**Phase:** 2 — Document Processing
**Status:** Not started

---

## Goal

Enable users to upload identity documents (driver's license, W-4, passport, etc.) directly in the chat interface. Use Active Storage to attach files to `Document` records, enforce file type and size constraints server-side, and provide a drag-and-drop (or button-click) upload UI in the React chat component. Each upload is linked to the user's `OnboardingSession` and the `Document` model (already in schema).

### Upload flow

1. Orchestrator reaches `document_upload` step → assistant asks user to upload a document
2. React renders `FileUpload` component (drag-and-drop zone + button)
3. User selects or drops a file
4. Client-side pre-validation: check file size and extension before sending
5. `POST /api/documents` with multipart form data (file + session_id + document_type)
6. Controller calls `Documents::UploadValidator` → rejects or proceeds
7. Active Storage attaches file to `Document` record
8. Document created with `status: "uploaded"`, linked to `OnboardingSession`
9. Controller returns JSON `{ id:, document_type:, status: "uploaded" }`
10. React displays success message in chat; channel notified so assistant can continue

### Allowed file types

| MIME type | Extension |
|-----------|-----------|
| `image/png` | .png |
| `image/jpeg` | .jpg, .jpeg |
| `application/pdf` | .pdf |

### Size limit

- **Maximum:** 10 MB per file
- Enforced both client-side (JS check before upload) and server-side (validator)
